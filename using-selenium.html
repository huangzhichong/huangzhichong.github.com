Ruby + Selenium framework integrated with Marquee

This framework is based on rspec (unit test framework in ruby) and selenium.

You can find the codes in here

File Structures

Common Folder

  • the common libs for loging, integration with marquee and configuration

log folder

  • log files

objects folder

  • collection of objects. Puts objects locator of one page into a ruby file.

  • Using a hash data struture to store the object locator

packates folder

  • An combinlation of objects and actions.

screeen_shots folder

  • tmp folder to put the screen shots when case fail

test_results folder

  • tmp folder to put html results generate by rspec

testsuites folder

  • automation scripts

config

  • configuration of Marquee , server ip and project name

main.rb

  • used for marquee client to kick off a testsuite

  • the command to start a testsuite is like

      ruby main.rb testsuit_name round_id browser

update_script_to_marquee.rb

  • ruby scritps to upload automation script information to marquee

Gemfile

  • all required gems are configure here

Get Started

  • setup the environment

    make sure you have ruby-1.9.2 installed, you can install it with RVM

      rvm install ruby-1.9.2

    make sure devkit is installed if you’re a windows user

    You need to install bundler to have gems configured with Gemfile

      gem install bundle

    Install all require gems with bundler( go to your directory)

      bundle install

  • Get the code

    Use git to clone the codes from git,

      git clone git://git.dev.activenetwork.com/aw-automation/rselenium-expert.git

    this will create a folder in your currect directory name as rselenium-expert

    if you don’t like this name, you can try below command,

      git clone git://git.dev.activenetwork.com/aw-automation/rselenium-expert.git folder_name

    you can login to git.dev.activenetwork.com with you email and domain password.

    then you need to follow the instruction to setup your ssh-key

    As we are going to automate searching in google home page, then let’s put this scenario into our script

    • put a it block into the run_script method, a it block is considered as a test case. you can have mutiple it blocks in the script

      you can describe what you want to do as string after it statement. this information will show in the test result html file. Also we are using this information as test case name to display in marquee

      the format of description is CaseID + "##" + Case Name

        def run_script
      it "Case01##Do the searching in the google home page" do
      end
  end

    • then, let’s add some test steps into scripts.

      • input search text on the google home page

      • click the search button on the home page

      i will add some codes for the steps like below,

        def run_script
      it "Case01##Do the searching in the google home page" do
          GoogleHome.input_search_text "Search Text"
          GoogleHome.click_search_button
      end
  end

    • we will going to create functions used in above script.

      adding file under package folder named Google_Home_Page_Functions.rb

      copy below codes into the file

        require "#{File.dirname(__FILE__)}/../common/operation"

  require "#{File.dirname(__FILE__)}/../objects/Google_Home_Object.rb"

  module GoogleHome

      class << self

          def input_search_text(string_text)
              $op.input GoogleHomePage.SearchText , string_text
          end

          def click_search_button
              $op.click GoogleHomePage.SearchButton
          end
      end
  end

    • we need to create object collections to support our function

      adding a file named Google_Home_Object.rbrb under objects

      adding below codes into the file

        module GoogleHomePage
    class << self   
      def SearchText
        resultHash=Hash.new
        resultHash["name"]="SearchText"
        resultHash["elementtype"]="WebEdit"
        resultHash["findtype"]="name"
        resultHash["findvalue"]="q"
        resultHash
      end

      def SearchButton
        resultHash=Hash.new
        resultHash["name"]="SearchButton"
        resultHash["elementtype"]="WebButton"
        resultHash["findtype"]="name"
        resultHash["findvalue"]="btnK"
        resultHash
      end
    end
  end
